DayAgent 移动端 — 技术方案

后端 Java + Python 全部复用，只做前端皮。选哪个技术路线？

A

Expo (React Native)

全新原生壳 · 推送通知 · 流畅动画 · OTA 热更新
推荐 — 最适合单人长期维护的项目

Pros

  • 原生推送（Expo Push）免费
  • 动画丝滑，60fps 粒子没问题
  • EAS Build 云端打包，不需要 Mac 也能出 iOS
  • TypeScript 全栈，API 层逻辑直接迁移

Cons

  • 像素风 UI 需要手写（但完全可控）
  • 需要重写全部组件（约 2-3 周）
  • 学习 Expo 生态（文件路由、expo-splash 等）
B

Capacitor (Vue 3 壳)

现有 Vue 代码包进原生容器 · 改造成本低 · 同构开发

Pros

  • 现有 Vue 组件直接复用，改动小
  • 一套代码跑 Web + Android + iOS
  • Capacitor 插件生态（相机、推送等）

Cons

  • Canvas 粒子/CRT 扫描线在手机上性能可疑
  • WebView 体验不如原生流畅
  • 像素风移动端适配工作量不亚于重写
  • 桌面端遗留代码（Electron IPC、窗口控制）成包袱
C

PWA

现有前端加 manifest.json · 浏览器安装 · 零额外依赖

Pros

  • 最快上线（1-2 天）
  • 不需要 App Store
  • 桌面/移动共用一套

Cons

  • iOS 不支持推送通知
  • 无法后台运行
  • 使用体验"像网页"而非 App
  • 每天两次推送提醒是最核心功能

我的推荐：A — Expo

理由很简单：每日规划助手最核心的价值是早 8 点推送规划 + 晚 9 点总结提醒。
PWA 做不到 iOS 推送，Capacitor 背着 Electron 遗留包袱在手机上很别扭。
Expo 重写工作量大，但写得干净——只做移动端需要的 3 个页面。
